<objective>
Package the required backend as a `docker compose up` stack — `redis` + `api` + `worker` — that runs with no host-side Trivy or Redis install, from a lean multi-stage `node:22-slim` image running as non-root. The worker is capped at `mem_limit: 200m` and launched under `--max-old-space-size=150`; the API entrypoint matches the assignment's verbatim self-test command. This is greenfield packaging (no Docker artifacts exist yet) satisfying OPS-01/OPS-02 (pulled forward per D-04).

Purpose: Make the stack reviewer-runnable from one command and set up the container the in-container OOM proof (Plan 03) runs against. The defining memory constraint (150MB heap under a 200m container) is encoded here; Plan 03 proves it empirically.

Output: `Dockerfile`, `.dockerignore`, `docker-compose.yml` at the repo root.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host Docker daemon ↔ worker container | The worker mounts `/var/run/docker.sock` to run Trivy as a sibling container (host-root-equivalent control) |
| image build ↔ registry | Base image `node:22-slim`, `redis:7-alpine`, and the pinned Trivy image are pulled from registries |
| compose network ↔ host | Only the API port is published; Redis stays internal |
| build context → image layers | Files copied into the image may leak secrets/planning if not ignored |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-02-01 | Elevation of privilege | `/var/run/docker.sock` mount on worker | high | accept | Locked user decision D-06 (single-tenant take-home, not production). Compensating controls: worker runs as non-root `node`, no added capabilities, ONLY the pinned `ghcr.io/aquasecurity/trivy:0.69.3` image is invoked, compose network is loopback/internal. Trade-off flagged for ONBOARDING (Phase 6, DOC-02) |
| T-05-02-02 | Information disclosure | secrets/`.env`/`.git`/planning in image layers | high | mitigate | `.dockerignore` excludes `.env*`, `.git`, `.planning`, `**/*.bundle`; multi-stage runtime drops build deps; `npm ci --omit=dev` (V12) |
| T-05-02-03 | Tampering | base + Trivy + Redis image supply chain | high | mitigate | Pinned tags: `node:22-slim`, `redis:7-alpine`, `ghcr.io/aquasecurity/trivy:0.69.3` (Docker Library + official ghcr); build via lockfile `npm ci` |
| T-05-02-04 | Information disclosure / EoP | unauthenticated Redis broker | medium | mitigate | Redis publishes NO host port — reachable only on the compose-internal network by api+worker |

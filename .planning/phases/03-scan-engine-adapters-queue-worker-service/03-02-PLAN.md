<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Worker adapter → git/Trivy/Docker subprocess | Repository and generated paths cross into external processes. |
| Host filesystem → report parser | Generated report files cross into the streaming parser. |
| Configuration → adapter paths | Validated temporary-root and engine settings control filesystem and subprocess behavior. |

## ASVS Level 1 Controls

- **Input and configuration validation:** Validate typed adapter inputs, generated scan identifiers, engine mode, `REDIS_HOST`, `REDIS_PORT`, and `SCAN_TMP_DIR` fail-closed before infrastructure use; reject invalid configuration rather than selecting an unsafe default.
- **argv, shell, and path safety:** Pass repository URLs, executable flags, and generated paths as discrete argv values with `shell:false`; never construct shell command strings, and confine clone/report/cache paths beneath the validated temporary root.
- **Bounded and redacted errors:** Persist only categorized, sanitized failure data capped at 500 characters; redact credentials and uncontrolled filesystem paths, while keeping detailed subprocess diagnostics out of Redis state.
- **Availability, Redis, and memory risks:** Write Trivy output to disk instead of buffering report JSON, bound diagnostics and subprocess behavior, use disposable/ephemeral artifacts, and preserve finite Redis/client retry and cleanup behavior so Redis or disk pressure cannot create unbounded memory or retry growth.

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-05 | Tampering/Elevation | git and Docker subprocess invocation | critical | mitigate | Use argv arrays, shell:false, generated paths under validated SCAN_TMP_DIR, no exec/string interpolation, read-only clone mount, and explicit /out report mapping. |
| T-03-06 | Denial of service | Trivy report output | high | mitigate | Force --output to disk, never buffer report stdout, and validate host report visibility before parsing. |
| T-03-07 | Information disclosure | subprocess diagnostics and scan errors | high | mitigate | Bound and redact persisted category/detail; retain detailed stderr only in logs. |
| T-03-09 | Availability | temporary artifacts | medium | mitigate | Centralize idempotent recursive cleanup and test success plus clone, Trivy, ENOSPC, and parser failure inputs. |
| T-03-SC | Tampering | npm package use | high | mitigate | Consume only the exact dependency pins approved by the Plan 01 preflight legitimacy gate. |
</threat_model>

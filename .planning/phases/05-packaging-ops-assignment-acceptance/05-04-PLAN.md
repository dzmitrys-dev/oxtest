# Plan 05-04 — Gap closure: docker-compose stack can actually scan (OPS-02 / Criterion #2)

<objective>
05-VERIFICATION.md found one blocking gap: the `node:22-slim` runtime image installed
no OS packages, so the worker's per-scan shell-outs — `git clone` (repo-cloner.adapter.ts)
and the `docker` CLI for the Trivy sibling (trivy-runner.adapter.ts, TRIVY_MODE=docker) —
failed ENOENT inside the image. CI stayed green because acceptance.mjs runs the worker on
the HOST (which has git+docker) and the OOM proof runs only the pure memtest, so nothing
exercised clone→scan inside the built image.

Fix approach (user-chosen Option A — keep D-05 non-root + D-06 docker-sibling): install
git + the docker CLI *client* in the runtime image (NOT the Trivy binary — Trivy stays a
sibling container so its RSS is off the 200m cap), make the non-root worker reach the
mounted socket robustly on any host, and add an in-image smoke test so the blind spot
cannot recur.
</objective>

<task id="1">
<name>Install git + docker CLI client in the runtime image</name>
<read_first>
- Dockerfile (runtime stage 35–50)
</read_first>
<action>
In the runtime stage, before the npm copy, add an apt layer: install `git ca-certificates`
plus `docker-ce-cli` from Docker's official Debian apt repo (client only — no daemon), then
clean apt lists. Client-only keeps Trivy in a sibling container (memory isolation) and the
image lean.
</action>
<acceptance_criteria>
- `docker build --target runtime` succeeds; `docker run <img> git --version` and
  `docker run <img> docker --version` both exit 0.
</acceptance_criteria>
</task>

<task id="2">
<name>Runtime entrypoint: reach the socket as non-root on any host</name>
<read_first>
- docker-compose.yml (worker service)
- Dockerfile (runtime stage tail)
</read_first>
<action>
Add docker-entrypoint.sh: starts as root, resolves the mounted /var/run/docker.sock's gid
at runtime, adds `node` to that group, then `exec setpriv --reuid node --regid node
--init-groups "$@"`. Dockerfile: COPY + chmod the script, drop `USER node`, set ENTRYPOINT.
Compose: remove group_add (entrypoint handles it), add `memswap_limit: 200m` (WR-02).
</action>
<acceptance_criteria>
- Running the image with the socket mounted: `id -u` == 1000 (non-root); a non-root
  `docker run ghcr.io/aquasecurity/trivy:0.69.3 --version` succeeds (sibling reachable).
- `docker compose config` valid; worker shows mem_limit==memswap_limit==209715200; no group_add.
</acceptance_criteria>
</task>

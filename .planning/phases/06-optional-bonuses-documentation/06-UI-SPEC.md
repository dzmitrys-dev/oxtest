## Color

One clean light theme. 60 / 30 / 10 split, plus meaningful semantic colors for severity and scan status (a security tool's status colors are informational, not decorative).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8fafc` (slate-50) | Page background |
| Secondary (30%) | `#ffffff` (white) | Cards, header bar, results table surface; borders `#e2e8f0` (slate-200) |
| Accent (10%) | `#4f46e5` (indigo-600), hover `#4338ca` (indigo-700) | See reserved list below |
| Destructive / Danger | `#dc2626` (red-600); soft `#fef2f2` (red-50) bg, `#b91c1c` (red-700) text | CRITICAL severity badge; Failed error card |

Text colors: primary `#0f172a` (slate-900); secondary/muted `#64748b` (slate-500); on-accent/on-danger `#ffffff`.

**Accent (indigo) reserved for:** the primary **"Start scan"** button (solid fill), input **focus ring**, and inline **hyperlinks** (e.g. the CVE `primaryUrl` link). Nothing else. Never use indigo for severity, borders, or body text.

### Scan status semantic colors (informational, not counted in the 10% accent budget)

| Status | Signal color | Treatment |

| Queued | slate-500 `#64748b` | Neutral pill + static dot |

## Screen Structure & Interaction Contract

Single centered column (`max-w-3xl`), one vertical flow:

1. **Header** â€” app title (Display) + subtitle (Label, muted). Static.
2. **Scan form card** (Secondary surface, `lg` padding): label â†’ URL input (full width, 40px) â†’ `Start scan` button. Client-side validation (Pattern 5) on submit and on blur; button disabled while empty/invalid or while a scan is in a non-terminal state.
3. **Status region** (renders below the form once a scan id exists) â€” a single card whose body is chosen by the `status` state machine. Show a compact status header line at top: `Scanning {owner/repo}` with the repo URL in mono + a status pill.

### Four-state renderer (D-07 â€” all four `ScanStatus` values MUST render; exact enum strings `Queued` / `Scanning` / `Finished` / `Failed`)

| State | Visual contract |
|-------|-----------------|
| `Queued` | Status pill (slate) + Queued heading/body. No spinner motion required beyond a static dot. Polling active. |
| `Scanning` | Animated inline SVG spinner (indigo) + Scanning heading/body. Polling active (2s). |
| `Finished` | Stop polling. Render count summary (red if n>0, emerald all-clear line if n===0) then the results table (or empty-state copy). |
| `Failed` | Stop polling. Red-50 error card: heading + humanized body + optional `error.detail` (mono, muted, wrapped, never truncated visually mid-word). |

Polling: stop on `Finished`/`Failed` (RESEARCH Pattern 4). While polling, the previously-rendered state stays visible (no full-card flash, no layout shift) â€” swap only the inner state block.

### Results table contract (D-08 â€” HARD: only stored `Vulnerability` fields; NO "Fixed version" column)

Columns, leftâ†’right (map only to the persisted shape `{ vulnerabilityId, pkgName, installedVersion, severity, title, primaryUrl }`):

| Header | Field | Rendering |
|--------|-------|-----------|
| Package | `pkgName` | Label size, mono |
| CVE | `vulnerabilityId` | Mono; hyperlink to `primaryUrl` (indigo link, opens new tab `rel="noopener noreferrer"`, trailing external-link SVG) |
| Installed | `installedVersion` | Mono |
| Severity | `severity` (always `CRITICAL`) | Red-50/red-700 pill badge, uppercase |
| Title | `title` | Body size, wraps; the flexible column |

Table styling: white surface, `slate-200` row dividers, semibold Label header row on a slate-50 header band, `sm`/`md` cell padding, zebra optional (very light slate-50 on odd rows). On narrow viewports the table may horizontally scroll within its card (do not collapse columns â€” the reviewer expects to see them). Do NOT add a "Fixed" / `fixedVersion` column â€” the field does not exist.

Empty `Finished` (0 rows): render the emerald all-clear empty-state copy instead of an empty table.

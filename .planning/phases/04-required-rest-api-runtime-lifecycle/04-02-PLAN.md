<objective>
Add runtime lifecycle: graceful shutdown on SIGTERM/SIGINT for both processes. The worker drains (finishes the active scan) with a bounded grace window before force-closing; both processes close their Redis handles cleanly via Nest lifecycle hooks. This closes the gap where an unbounded `worker.close()` would hang past Docker's SIGKILL window and where the raw `useFactory` ioredis client is never `.quit()`ed.

Purpose: Deliver success criterion #4 (ROADMAP Phase 4) and ERR-05. Shutdown correctness is a Phase 5 packaging prerequisite (the container inherits a clean SIGTERM→exit path within the 10s stop grace).

Output: A pure `raceDrain()` (unit-tested), a `WorkerShutdown` lifecycle provider (`@nestjs/bullmq`-adjacent, validated only by the compiled harness in Plan 03), an `onModuleDestroy` on the shared repository adapter, and the `SHUTDOWN_GRACE_MS` env key.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS signal → process (SIGTERM/SIGINT) | External (Docker/orchestrator) signal crosses into the process; must produce a clean, bounded exit. |
| process → Redis (connection teardown) | Open ioredis/BullMQ handles must be closed so the process exits without needing SIGKILL. |
| env → config (SHUTDOWN_GRACE_MS) | Operator-supplied value; must be schema-validated fail-closed. |

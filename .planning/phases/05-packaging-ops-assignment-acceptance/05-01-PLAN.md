<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API (`POST /api/scan`) | Untrusted `repoUrl` crosses into validation, enqueue, and log lines |
| worker → stdout logs | Scan lifecycle detail (repoUrl, failure reason) is serialized to ndjson read by operators/CI |
| app ↔ Redis | ioredis client whose connection can drop at any time |
| npm registry → build | `pino`/`pino-pretty` install into the dependency tree |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01-01 | Information disclosure | pino log lines (engine + enqueue) | high | mitigate | Log ONLY `scanId`/status/bounded reason via structured pino fields; never log Trivy report bodies or raw repo content (V7); failure `detail` stays capped at 500 chars (existing classifier) |
| T-05-01-02 | Tampering | log records built from untrusted `repoUrl` | medium | mitigate | Pass `repoUrl` as a structured pino field object, never string-interpolated into the message (log-injection guard, V7); repoUrl is already host-allowlisted |
| T-05-01-03 | Tampering | `GithubUrlPipe` (WR-01) | high | mitigate | Return the canonical `https://github.com/{owner}/{repo}` so the enqueued/cloned string equals the validated form, closing the validate-vs-use parser differential (V5) |
| T-05-01-04 | Denial of service | shared `REDIS_CLIENT` (WR-03) | high | mitigate | Attach a non-throwing `error` listener so a Redis drop cannot raise an unhandled 'error' and crash the process (V7) |
| T-05-01-05 | Denial of service | `SHUTDOWN_GRACE_MS` bound (WR-02) | medium | mitigate | Cap Joi `.max()` at 9000 so an in-flight scan always drains before Docker's 10s SIGKILL rather than being hard-killed (V14) |
| T-05-01-06 | Elevation / Tampering | shipped pino transport (worker thread) | medium | mitigate | No transport in production/container/CI (D-04b); `pino-pretty` is a devDependency gated behind `NODE_ENV==='development'`, never in the image — also protects the graded RSS budget |
| T-05-01-SC | Tampering | `pino`/`pino-pretty` supply chain | high | mitigate | Both `OK` in the RESEARCH Package Legitimacy Audit (official pinojs org, no postinstall); pinned exact versions installed via lockfile `npm ci`; pino already resident transitively via Fastify |
</threat_model>

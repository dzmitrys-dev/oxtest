## Trust Boundaries

| Boundary | Description |
|---|---|
| Integration harness → Docker/Redis | Test-controlled subprocesses and disposable services receive generated paths and ports. |
| Worker process → fixture repository | Git clone and Trivy operate on committed test content. |
| CI runner → package/image registries | CI downloads npm packages and the pinned Trivy image. |
| Test assertions → Redis state | Harness reads persisted domain state and failure details. |

## ASVS Level 1 Controls

Apply OWASP ASVS Level 1 controls to every boundary in this plan: V5/V14 require validation of harness arguments, environment, and probe results; V5/V8 require argv arrays, `shell:false`, and confined disposable paths/ports for Docker, Git, and worker processes; V7/V8 require bounded, redacted CI diagnostics and no raw credentials or uncontrolled paths in Redis assertions; V12/V14 require bounded polling/timeouts, disposable-resource cleanup, deterministic fixture size, explicit Redis availability handling, and fail-closed unknown feasibility states.

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-10 | Tampering | Integration child-process commands | high | mitigate | Spawn with argv arrays and shell:false, validate generated temp paths, and use explicit cleanup traps that preserve the primary exit status. |
| T-03-11 | Denial of service | Docker Trivy/Redis integration | medium | mitigate | Use disposable containers, bounded polling/timeouts, one worker concurrency, and deterministic fixture size; never depend on live GitHub content. |
| T-03-12 | Information disclosure | CI logs and failure assertions | medium | mitigate | Redact credentials and uncontrolled paths from persisted reasons; print only bounded diagnostics and container identifiers. |
| T-03-13 | Supply chain | npm and Trivy image downloads | high | mitigate | Use lockfile npm ci, approved exact packages, reviewed official image `ghcr.io/aquasecurity/trivy:0.69.3`, and fail the process contract if the image tag or mount policy changes. |
| T-03-SC | Tampering | CI package installation | high | mitigate | The dependency legitimacy approval is a prerequisite in Plan 01; CI consumes the committed lockfile rather than resolving floating versions. |
</threat_model>
